Dialog Flow

1
CLU Recognition — Detect BookFlight intent + pre-fill any mentioned entities
2
Origin Prompt — If or_city is missing, ask "Where from?"
3
Destination Prompt — If dst_city is missing, ask "Where to?"
4
Departure Date — Parse date using Recognizers-Text library
5
Return Date — Validate return is after departure
6
Budget Prompt — Accept currency amounts
7
Confirmation — Summary card, Yes/No → record telemetry

Application Insights Monitoring

📡 Custom Events Tracked

EventTrigger
LuisIntentEvery user message
BookingCompletedUser confirms booking
BotComprehensionFailureBot can't understand
DialogCancelledUser cancels flow

🔔 Alert Rule (KQL)

customEvents | where name == "BotComprehensionFailure" | summarize count() by bin(timestamp, 5m) | where count_ >= 3

→ Triggers email alert when >3 failures in any 5-minute window

Monitoring Methodology

📏 Evaluation Criteria

  • Intent F1 ≥ 0.85
  • Entity F1 ≥ 0.80 per entity
  • Comprehension failure rate < 10%
  • Monthly precision/recall trend tracking

🔄 Retraining Triggers

  • Emergency — >20% failure rate (24h)
  • Routine — F1 drops below threshold (2 weeks)
  • Scheduled — Full retrain monthly

🔍 Production Monitoring Flow

1
User messages → Bot logs intents + entities to App Insights
2
BotComprehensionFailure events accumulate in real time
3
KQL alert fires → email notification to team
4
Weekly review → retrain if thresholds breached
